HOW TO TEST RANDOM FOREST AGRICULTURE FEATURES
================================================================================

METHOD 1: Interactive Test (Recommended)
-----------------------------------------

1. Open RStudio

2. Copy and paste this ONE LINE into the R console:

   source("C:/Users/carly/Downloads/Rflow worked version 3/Rflow/LOAD_AND_TEST.R")

3. Press Enter and watch it run!

   This will:
   - Install dependencies (devtools, roxygen2, randomForest, ggplot2)
   - Generate documentation
   - Load the package with devtools::load_all()
   - Run tests on all 5 agriculture applications
   - Show you beautiful plots
   - Give you example code to use


METHOD 2: Manual Step-by-Step
------------------------------

Open RStudio and run these commands one by one:

# 1. Set working directory
setwd("C:/Users/carly/Downloads/Rflow worked version 3/Rflow")

# 2. Install dependencies
install.packages(c("devtools", "roxygen2", "randomForest", "ggplot2"))

# 3. Generate documentation
roxygen2::roxygenize()

# 4. Load package
devtools::load_all()

# 5. Quick test
soil_data <- generate_ag_data("soil", n = 150)
result <- rf_soil_analysis(soil_data, "fertility", "classification")
plot_rf_importance(result)


METHOD 3: Run Full Demo
------------------------

After loading the package (Method 2, steps 1-4), run:

source("examples/random_forest_agriculture_demo.R")

This runs a comprehensive demo of ALL agriculture applications.

Solution:

  devtools::load_all()

Error: "cannot open file 'LOAD_AND_TEST.R'"

NEXT STEPS
----------

After testing, you can:

1. Use with your own data:
   my_data <- read.csv("your_data.csv")
   result <- rf_soil_analysis(my_data, "your_target", "classification")

2. Save your trained models:
   saveRDS(result$model, "my_model.rds")

3. Load and use later:
   model <- readRDS("my_model.rds")
   predictions <- predict(model, newdata = new_data)

4. Read full documentation:
   - RANDOM_FOREST_QUICKSTART.md
   - RANDOM_FOREST_AGRICULTURE.md
